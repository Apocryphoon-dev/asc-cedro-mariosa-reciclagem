## 🧠 Como Ajudar com Desenvolvimento Web (Sites)

### Arquitetura e Organização
- **Separação de responsabilidades**: separar marcação (HTML), estilos (CSS) e comportamento (JS). Para apps, organizar em componentes claros (UI, serviços, roteamento)
- **Modularização**: CSS modular (BEM, CSS Modules, Tailwind utilities), componentes reutilizáveis
- **Estrutura de projeto**: suportar static sites, SSG/SSR (Next.js, Astro) ou SPA conforme necessidade

### Acessibilidade (a11y)
- **Semântica HTML**: use tags corretas (`nav`, `main`, `header`, `footer`, `button`, `form`)
- **Keyboard first**: navegação por teclado garantida
- **ARIA quando necessário**: preferir HTML semântico antes de ARIA
- **Contraste e legibilidade**: cores e tamanhos acessíveis

### Responsividade e Design
- **Mobile-first**: pensar estilos por largura menor primeiro
- **Layout fluido**: usar unidades relativas (%, rem, vw) e media queries
- **Imagens responsivas**: `srcset`, `sizes`, formatos modernos (WebP/AVIF)

### Performance e Otimização
- **Critical CSS**: reduzir bloqueio de renderização
- **Lazy loading**: imagens, iframes e módulos JS não críticos
- **Pré-conexões e cache**: `preconnect`, `preload`, cabeçalhos HTTP (Cache-Control)
- **Minificação e bundling**: otimizar assets, tree-shaking

### SEO e Conteúdo
- **HTML semântico e metatags**: título, description, Open Graph, canonical
- **Performance influencia SEO**: tempo de carregamento e Core Web Vitals
- **Sitemaps e robots**: expor sitemap.xml e robots.txt corretamente

### Segurança para Sites
- **Validação e sanitização**: evitar XSS em conteúdos dinâmicos
- **Cabeçalhos de segurança**: CSP, HSTS, X-Frame-Options
- **Secrets fora do repo**: usar variáveis de ambiente no build/host

### Ferramentas e Integrações
- **Hospedagem/Deploy**: Vercel, Netlify, GitHub Pages, serviços compatíveis com SSG/SSR
- **CDN e assets**: usar CDN para assets estáticos
- **CMS e Headless**: integração com Contentful, Sanity, Strapi, Netlify CMS quando necessário

### Testes e Qualidade
- **Testes visuais e unitários**: Jest, Playwright, Cypress para E2E
- **Linting e formatação**: ESLint, Stylelint, Prettier
- **Auditoria de performance**: Lighthouse / PageSpeed Insights

### Deploy e Observabilidade
- **Monitoramento**: Sentry/Raygun para erros, ferramentas de analytics para uso e desempenho
- **Logs e alertas**: logs do servidor e erros críticos com contexto

## 🛠️ Como Se Comunicar Comigo

Quando sugerir código ou mudanças para um site:
- ✅ Forneça o **exemplo completo** de HTML/CSS/JS necessário para reproduzir o comportamento
- ✅ Inclua **trechos comentados** quando houver lógica não trivial
- ✅ Use **nomes semânticos** para classes, ids e componentes
- ✅ Para trechos assíncronos em JS/TS: incluir `try/catch` e tratamento de erros
- ✅ Aponte claramente impactos em performance e acessibilidade
- ✅ Sugira como testar manualmente e via automação
- ❌ Evite soluções proprietárias que prendam o projeto a um vendor sem necessidade
